定义

FSOP 是 File Stream Oriented Programming 的缩写，根据前面对 FILE 的介绍得知进程内所有的_IO_FILE 结构会使用_chain 域相互连接形成一个链表，这个链表的头部由_IO_list_all 维护。

FSOP 的核心思想就是劫持_IO_list_all 的值来伪造链表和其中的_IO_FILE 项，通常来说一般是直接利用任意写的漏洞修改_IO_list_all直接指向可控的地址。但是伪造后又该如何触发呢？

FSOP 选择的触发方法是调用_IO_flush_all_lockp，这个函数会刷新该函数的功能是刷新所有FILE结构体的输出缓冲区，相当于对每个 FILE 调用 fflush，也对应着会调用_IO_FILE_plus.vtable 中的_IO_overflow。相关源码如下，文件在libio\genops中：

输出缓冲区的数据保存在fp->_IO_write_base处，且长度为fp->_IO_write_ptr-fp->_IO_write_base，因此上面的if语句实质上是判断该FILE结构输出缓冲区是否还有数据，如果有的话则调用_IO_OVERFLOW去刷新缓冲区(刷新缓冲区指的是输出时将输出缓冲区里面的内容传给屏幕，显示出来；将输入缓冲区里面的内容传给变量。达到 清空缓冲区的效果)。可以看出来该函数的意义是为了保证数据不丢失，因此在程序执行退出相关代码时，会去调用函数去刷新缓冲区，确保数据被保存。

而_IO_flush_all_lockp 不需要攻击者手动调用，在一些情况下这个函数会被系统调用：

  1. 当 libc 执行 abort 流程时
  2. 当执行 exit 函数时
  3. 当执行流从 main 函数返回时

利用

利用的方式为：伪造IO FILE结构体，并利用漏洞将_IO_list_all指向伪造的结构体，或是将该链表中的一个节点（_chain字段）指向伪造的数据，最终触发_IO_flush_all_lockp，绕过检查，调用_IO_OVERFLOW时实现执行流劫持。

漏洞利用

泄露出puts函数的地址，我们可以得到libc的基址，进而可以算出_IO_2_1_stderr的地址.我们可以利用add函数的溢出漏洞修改page的contentchunk_addr 为_IO_2_1_stderr。这样我们就可以控制其内容，在里面进行伪造了。

为了调用_IO_flush_all_lockp中的_IO_OVERFLOW。我们需要绕过一些check，即满足fp->_mode <= 0和fp->_IO_write_ptr > fp->_IO_write_base。因此我们可以设置_IO_2_1_stderr_.file._mode为0、_IO_2_1_stderr_.file._IO_write_base为0、_IO_2_1_stderr_.file._IO_write_ptr为1.然后伪造一个fakevtable(设置__overflow的值为system，因为_IO_flush_all_lockp会调用_IO_OVERFLOW），让`_IO_2_1_stderr.vtable指向该fake_vtable。写入IO_2_1_stderr时将 /bin/sh 写到_IO_FILE的头部，调用虚函数时_IO_FILE` 是第一个参数。
