house-of-pig学习

发表于2023-04-03|更新于2023-04-09
|阅读量:

house of pig 是通过 libc2.31 下的 largebin attack以及 FILE 结构利用，来配合 libc2.31 下的 tcache stashing unlink attack 进行组合利用的方法。主要适用于程序中仅有 calloc 函数来申请 chunk，而没有调用 malloc 函数的情况。该方法最为核心的地方在于，利用了 glibc 中 IO_str_overflow 函数内会连续调用 malloc 、memcpy、free 函数的特点，并且这三个函数的参数都可以由 FILE 结构内的数据来控制。
附件下载：https://github.com/01dwang/house_of_pig/blob/main/pig.zip

glibc2.31下的_IO_str_overflow利用

漏洞分析

根据前文的分析可得，程序在add时会将flag1和flag2标志位置0，在show和edit功能中会检查第一个标志位flag1是否为0，为0的情况下才能够进行相关操作；delete功能会检查flag1和flag2这两个标志位是否都为0，都为0才会进行free，free之后将flag1和flag2标志位置1，但并没有清空堆指针，所以这里可能会存在uaf。

接下来是角色切换流程。

如果切换的角色和当前的角色不同，那么我们就要在global_pigs保存当前的house_message结构体。

但是注意到这里我只保存了chunk_ptr，size，flag2，并没有保存flag1

而在后面的恢复函数中，恢复global_pig中保存的chunk_ptr，size，flag1，flag2，而global_pig中的flag1值为0。

也就是说，我们可以先使用peppa释放一个chunk，此时flag1、flag2标志位为1，接着我们切换角色为Mummy，此时会保存peppa的chunk_ptr、size、flag2，然后再切换角色为peppa，就会将global_house中保存的chunk_ptr、size、flag1，flag2恢复给peppa，那么peppa的flag1=0，flag2=1。除了无法进行delete，show和edit都可使用，即存在一个UAF。

漏洞利用

本题的利用流程主要如下：

  • 首先泄露libc基址和堆地址

  • 利用largebin attack在free_hook-8处写一个堆地址

  • 利用largbin attack，在_IO_list_all中写入一个堆地址

  • 利用tchache stashing unlink attack，将free_hook-0x10链入tcache中,并在_IO_list_all指向的堆中布置fake FILE。

  • 执行exit时触发exit-> __run_exit_handlers->_IO_cleanup->_IO_flush_all_lockp，调用我们布置的fake FILE中伪造的vtable 中的_IO_str_overflow，利用里面的malloc将free_hook-0x10申请出来，再利用memcpy将system函数写到free_hook上，最后利用free释放掉一个/bin/sh\x00，即可getshell。

首先为tchache stashing unlink attack做准备

由于本程序采用calloc分配chunk，那么我们如何在tcache 中布置好5个chunk，同时在small bin 中布置好两个相同大小的chunk呢？

我们可以先释放5个0xa0的chunk到tcache中，然后创建8个0x160的chunk并释放，使chunk_A0进入unsorted bin，然后创建一个0xc0的chunk，分割0x160的unsorted bin，剩余0xa0的unsorted bin

然后再创建一个0x190的chunk，使unsorted bin进入small bin中，接着在创建7个0x190的chunk并释放，使chunk_A8进入unsorted bin 中，同时使用相同的方法对unsorted bin进行分割，并使剩余的0xa0进入small bin。

泄露libc和heap_addr

先创建一个chunk，防止chunk_A16与top_chunk合并，然后释放chunk_A16，使其进入unsorted bin。接着创建一个0x450的chunk，使unsorted bin进入large bin。

然后show(16)泄露libc地址，填充chunk_A16的fd和bk为'A'*0x10，再次利用show(16)泄露堆地址。

最后要恢复chunk_A16的fd和bk。

利用largebin attack在free_hook-8处写一个堆地址

首先创建三个0x440的chunk用于后续利用，删除chunk_B8使其进入unsorted bin，并创建一个0x460的chunk使chunk_B8进入large bin，删除chunk_A17使其进入unsorted bin。

接着修改chunk_B8的bk_nextsize为free_hook-0x28。

然后分配一个0xb0的chunk 触发large bin attack，当我们分配0xb0的chunk时，tcache中没有合适的chunk，于是先将unsorted bin放入largebin 中，发现其大小足够，再对其进行切割，产生remainder，将last_remainder插入到unsorted bin中。

最后注意恢复chunk_B8的fd_nextsize和bk_nextsize。

可以看到free_hook-0x8写入了一个堆地址

利用largbin attack，在_IO_list_all中写入一个堆地址

首先将上述分割的unsorted bin分配走，接着利用上述的思路在_IO_list_all中写入一个堆地址，最后注意恢复chunk_B8的fd_nextsize和bk_nextsize。

进行tcache_stashing_unlink 攻击，同时在_IO_list_all指向的堆中布置fake FILE

chunk_A8在上文中被分割为一个0xF0和0xA0的chunk，因此我们可以利用chunk_A8修改0xA0的small bin 的bk指针，同时也要注意fd指针的修改，使其指向原本的chunk。

总结

house of pig 是通过 libc2.31 下的 largebin attack以及 FILE 结构利用（_IO_str_overflow中的malloc、memcpy、free），来配合 libc2.31 下的 tcache stashing unlink attack 进行组合利用的方法。

适用版本：glibc 2.23-至今

在glibc2.21-glibc 2.33时我们可以设置好fake_FILE结构体的一些字段，利用memcpy将free_hook修改为system，并设置old_buf = fp->_IO_buf_base为/bin/sh。free(old_buf)即可getshell。

如果开启沙盒

我们如果事先把 malloc_hook 改为 setcontext +0x61的地址，并且提前布置好 ORW 的位置（把 srop_addr 放在 _IO_write_ptr 上，即rdi+0x28处），那么再调用 _IO_str_overflow 的时候就会先控制好 rdx 寄存器，再通过 malloc_hook 来触发 setcontext 来进行对 rsp ，rip 的控制，从而控制程序执行流来执行 ORW。

具体利用可参考：https://www.anquanke.com/post/id/216290#h3-8

在glibc 2.34之后，__free_hook，__malloc_hoook，__realloc_hook这些函数都被删除了。

那么我们该如何利用house of pig呢？注意到在_IO_str_overflow中调用链memset函数，memset函数在libc中含有got表，并且可写。因此我们可以把原先的house of pig的改写__free_hook转为改写memset在libc中的got表。
